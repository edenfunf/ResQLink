# Security and Limitations

本文件說明 災鏈 ResQLink 的定位、資料倫理與目前限制，供開源使用者與比賽審查參考。
請在採用、展示或延伸本專案前完整閱讀。

## 1. 非官方系統聲明

災鏈 ResQLink 是**公民科技（civic tech）輔助工具**，並非任何政府機關之官方系統，
亦未獲任何主管機關背書或授權。本專案僅供研究、教育、競賽與技術展示用途。

## 2. 不取代政府災害應變指揮

本系統**不取代、不等同於**政府之災害應變指揮、預警發布或官方公告。
任何疏散、撤離、救援與安全決策，**一律以政府官方公告與現場指揮為準**。
請勿將本系統之任何輸出，作為即時人身安全判斷的唯一依據。

## 3. 個資（PII）處理限制

民眾通報可能包含個人資料（姓名、聯絡方式、地點）。本專案在比賽 / 展示階段**以最小化方式處理**，
但**尚未達到正式環境（production）的個資保護標準**。採用前必須補齊第 4 點所述控制措施。

## 4. reporter_contact / provider_contact 目前為明文儲存

`disaster_reports.reporter_name` / `reporter_contact` 與 `resource_offers.provider_name` /
`provider_contact` 目前**以明文儲存於資料庫**，僅為簡化展示。對外列表（reports / resources）與
GeoJSON 一律不輸出聯絡方式，但 production **必須**：

- **加密**：對 PII 欄位做靜態加密（encryption at rest）。
- **權限控管**：以 RBAC / 認證限制誰能讀取聯絡方式（目前無登入、無權限系統）。
- **遮罩**：在非必要情境（列表、統計）對 PII 做遮罩（masking）。
- **保存期限**：設定資料保存與刪除政策（資料最小化）。

## 5. Public Preview approved-only 設計

所有自動生成的元件預設為 `pending_review`，採**白名單式公開**。
`GET /v1/public/preview/{slug}` **只回傳 `approved`** 的內容，
未經審核 / 退回 / 封存的內容對外**完全不可見**，且前端無法繞過此規則。
此設計用以降低未經查核之資訊外流的風險。

## 6. GeoJSON 不輸出 PII

`GET /v1/incidents/{id}/reports.geojson` 為對外可被任意地圖系統取用的圖層，
其 `properties` **永不包含** `reporter_name` / `reporter_contact`，
僅輸出救災調度所需的最小資訊（需求類型、嚴重度、狀態、地點描述）。
PII 僅在需授權的後台 API（`GET /v1/reports/{id}`）出現，正式環境須再加權限控管。

## 7. AI 使用狀態

預設採 **rule-based generation**（規則式生成）。另提供**選用（opt-in）的 AI 草擬層**：
呼叫 Bootstrap 時帶 `use_ai=true`，會以 LLM **並行草擬部分自由文字欄位**（公告內容、入口標題、
表單引導文字），其餘結構仍由規則式產生。設計界線如下：

- **AI 只生文字、不生結構**：欄位、流程與風險分級仍是確定性的規則式，可逐項稽核。
- **永不自動公開**：AI 產出與規則式產出一樣，一律標為 `pending_review`，須經人工審核閘門才公開。
- **不碰個資**：民眾姓名 / 聯絡方式不會送入模型。
- **保底**：任一欄位的 AI 草擬若失敗或未設定金鑰，自動退回該欄位的規則式內容。
- **不參與安全決策**：AI 不做疏散 / 撤離 / 調度判斷，安全資訊以政府官方公告為準。

金鑰以環境變數（`OPENAI_API_KEY`）提供、不進版控；未設定時系統完全以規則式運作。

## 7.5 對外發布為模擬連接器

`POST /v1/artifacts/{id}/publish`（FB / LINE 發布）目前使用**模擬連接器（simulated）**：
它會記錄發布動作並寫入 outbox，但**不會真的對外發文**。設計界線：

- **僅限審核通過**：只有 `approved` 的元件才能發布，未審核內容無法被推送出去（維持審核閘門）。
- **不實際外連**：未設定真實憑證前，不對 Facebook / LINE 發出任何請求。
- 正式環境需以憑證接上真實 API，並補上發布權限控管與速率限制。

## 8. 未串接真實官方 API

本專案提供**開放資料連接器**（`/v1/connectors`）：可將**中央氣象署（CWA）地震報告**、
**NCDR 災害示警（CAP）**等官方開放資料映射成標準事件，依 `source_ref` 冪等去重。
- **離線可用**：未設定授權碼時，以內建範例 payload（`/demo`）或外部提供的 payload（`/ingest`）建立事件，方便展示。
- **即時同步**：設定 `CWA_API_KEY`（opendata.cwa.gov.tw 免費申請）後，`/sync` 可實際抓取官方資料。
- 範例資料仍為**展示性質**，請勿視為官方即時警報；正式營運須接妥憑證、驗證來源可信度並標註。
- `data.gov.tw` 列為資料集來源（避難所、地理圖資等加值），非事件警報來源。

## 9. 災害資訊須人工審核

任何要對外公開的內容，均須經由 Review Task 的**人工審核**通過後才會顯示。
本專案刻意不自動公開生成內容，以避免錯誤或過期資訊在災時造成資源錯置或民眾風險。

## 10. 不應把歷史或模擬資料當成即時安全判斷依據

本專案之範例（samples）、seed demo 與任何展示資料均為**模擬 / 歷史性質**，
**不代表即時災情**。請勿據此做出任何即時的人身安全或疏散決策。

## 11. 未來 production 建議

若要將本專案推進到正式營運，建議至少補齊：

- **auth / RBAC**：登入、角色與權限控管。
- **audit logs review**：審核與存取的稽核軌跡與定期檢視。
- **encrypted PII**：個資靜態加密與金鑰管理。
- **official data connector**：以 `source_refs` 接上真實官方資料來源並標註可信度。
- **rate limiting**：通報與公開 API 的流量限制與濫用防護。
- **backup / disaster recovery**：資料備份與災難復原計畫。

---

> 一句話：災鏈 ResQLink 讓災害發生的第一個小時更快上線，但它是**輔助工具**，
> 任何安全決策請以官方為準。
